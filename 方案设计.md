> 版本：v0.2  
> 日期：2026-07-02  
> 说明：本方案为“不动手”阶段的设计文档。基于对两个源项目的源码级分析，给出以 `lvsim`（火箭仿真程序）为基础、吸收 `ballistics`（外弹道方程）能力的整体架构、融合策略、开发阶段与 GitHub 交付计划。

## 1. 目标与定位

### 1.1. 最终产品形态

在 `D:\程序\弹道模拟程序` 中构建一个**统一的多域弹道仿真平台**，按飞行环境明确划分为两大仿真域：

#### 1.1.1. 域 I：大气层内仿真

| 任务 | 来源基础 | 关键能力 |
|---|---|---|
| **炮弹 / 无动力弹道** | `ballistics` | MPM、6-DOF、G1/G7 阻力、风偏、火控反解、射表验证 |
| **导弹 / 有动力弹道** | 两者融合 | 3-DOF/6-DOF、推力-高度修正、阶段分离、命中精度、热流 |

#### 1.1.2. 域 II：大气层外仿真

| 任务 | 来源基础 | 关键能力 |
|---|---|---|
| **运载火箭入轨** | `lvsim` | 多级并联/串联、PEG 闭环制导、入轨配平、Δv 预算 |
| **洲际弹道导弹（ICBM）** | 两者融合 | 主动段（lvsim）+ 无动力中段 + 再入段（ballistics） |
| **亚轨道/探空火箭** | 两者融合 | 动力上升 + 无动力弹道 + 落地/回收 |

#### 1.1.3. 跨域能力

| 能力 | 来源 | 说明 |
|---|---|---|
| **可视化** | 两者融合 | 飞行轨迹、地面航迹、三维地球、轨道、再入、落点散布、动画 |
| **教学/分析工具** | 两者融合 | 蒙特卡洛、工况对照、CEP、Δv 预算、CLI/GUI/API 三入口 |

### 1.2. 核心设计原则

1. **以 `lvsim` 为工程骨架**：保留其包结构、阶段（phase）驱动积分、零件库装配、可视化风格。
2. **以 `ballistics` 为物理增强库**：吸收其 MPM、6-DOF、ECI、风场、地形、射表验证、事件检测。
3. **统一转换层，而非统一坐标系**：每个 `dynamics` 模块声明自己的原生坐标系；域 I 默认 ENU，域 II 默认 ECI；转换函数统一放在 `frames.py`。
4. **可插拔模型**：大气、重力、风、阻力、制导律、控制律均可注入。
5. **可视化作为一等公民**：v1 先保证 CLI 静态出图，GUI 作为阶段 4 可选增强，Web 放第二阶段。
6. **渐进式迁移**：先跑通“火箭动力上升 + 无动力弹道落地”这一最小闭环，再扩展其他域。

### 1.3. 根本性架构决策：复制迁移 vs 依赖包装

在动手前必须显式选择新平台与两个源项目的代码关系。存在两条路线：

| 路线 | 做法 | 优点 | 缺点 |
|---|---|---|---|
| **A. 依赖包装** | 把 `lvsim` 和 `ballistics` 作为本地可编辑包安装；新平台只写适配层、统一配置、统一入口、新增可视化。 | 源项目修 bug / 加功能时，新平台自动受益；代码量最小；分叉最少。 | 两个包接口风格差异大，适配层可能很厚；版本号/状态对象不统一；某些内部函数未暴露。 |
| **B. 复制迁移**（本文默认） | 将需要的源码文件复制/重构进新包 `ballistic_sim/`，按需改写。 | 接口完全可控；可自由统一命名、状态向量、配置层；无外部包版本耦合。 | 形成三份分叉代码；源项目后续改进不会自动同步；迁移工作量大。 |

**本方案选择路线 B（复制迁移）**，理由：
1. 两个源项目的状态向量、配置对象、命名风格差异显著，适配层成本可能高于直接重构。
2. 目标是将两者“深度融合”成一个统一平台，而非在之上做薄包装。
3. 两个项目均为用户自有代码，迁移时可直接修改内部实现。

**纪律要求（迁移期间）**：
- 阶段 0 起，对两个源项目中“即将被迁移的模块”实施**冻结**：不再在源项目里修改这些模块的接口与核心算法。若发现 bug，优先在新平台修复，并反向评估是否同步回源项目。
- 维护一份 `docs/migration_scope.md`，明确哪些模块已迁移、哪些仍依赖源项目、哪些被弃用。
- 阶段 5 完成后，新平台成为唯一维护目标；源项目保留为历史参考，或逐步归档。

**可回退措施**：
- 每完成一个阶段打 git tag；
- **阶段 2 两个 MVP 跑通时同步上线迁移对拍脚本**（见 §8.4 黄金文件方案），作为后续阶段的回归门禁；
- 对拍脚本在本地用于生成黄金文件，在 CI 中只对比新平台输出与已提交的黄金文件。

## 2. 两大仿真域划分

根据任务特性，将整个平台明确划分为**大气层内仿真域**与**大气层外仿真域**。两域在物理模型、坐标系侧重、可视化重点上有显著差异，但通过统一的 `Phase` 阶段驱动与 `SimConfig` 配置层无缝衔接。

---

### 2.1. 域 I：大气层内仿真

适用于飞行全程或主要飞行段位于地球稠密大气层内（通常 `h < 86 km`）的对象，细分为两类：

#### 2.1.1. 炮弹 / 无动力弹道

- **典型对象**：榴弹、迫击炮弹、坦克穿甲弹、增程弹、制导炮弹。
- **动力学模型**：
  - 默认 **MPM（修正质点模型，8 维）**：快速、适合火控反解与蒙特卡洛。
  - 可选 **6-DOF 刚体模型（13 维）**：用于姿态、稳定性、鸭舵控制分析。
- **关键物理**：
  - G1/G7/自定义阻力律；
  - 攻角诱导阻力、升力、马格努斯力；
  - 自转、陀螺稳定因子；
  - 风偏（均匀风/廓线风/Dryden 阵风）；
  - 科氏力、地球曲率（远程）。
- **终止条件**：几何高度 ≤ 地形高度（或 `h ≤ 0` 简化模式）。
- **核心输出**：射程、横偏、落角、飞行时间、落点散布、CEP、稳定因子。

#### 2.1.2. 导弹 / 有动力弹道

- **典型对象**：战术导弹、巡航导弹、防空导弹、火箭增程弹。
- **动力学模型**：
  - 默认 **3-DOF 质点 + 推力 + 气动（7 维）**：适合快速任务级分析。
  - 可选 **6-DOF**：用于控制回路、机动过载分析。
- **关键物理**：
  - 推力随高度变化（背压修正）；
  - 燃料消耗与质量变化；
  - 阶段分离/助推器脱落；
  - 攻角程序/过载程序制导；
  - 气动加热（再入时）。
- **终止条件**：命中目标距离阈值、燃尽后转入无动力弹道、落地。
- **核心输出**：命中精度、飞行时间、剩余速度、最大过载、热流峰值。

#### 2.1.3. 大气层内统一仿真流程

```
配置 SimConfig
    └─> mission = "projectile" / "missile"     # 域 I：大气层内
        ├─> vehicle.type 由 mission 推导
        ├─> environment.atmosphere = 扩展 ISA
        ├─> environment.wind = 廓线 / Dryden / GRIB2
        ├─> environment.terrain = 平面 / DEM
        └─> guidance = 无 / 程序攻角 / 火控反解

build_phases(cfg)
    └─> [Ascent/PoweredPhase] （导弹有动力段，可选）
        [CoastingPhase]      （无动力滑翔/下降段，可选）
        [TerminalPhase]      （落地/命中事件）

simulate()
    └─> 按 fidelity 选择 dynamics: mpm / sixdof / point_mass
        事件链：apogee / burnout / target_distance / impact_ground

postprocess()
    └─> 射程、落角、落点、CEP、过载、热流、风偏贡献
```

---

### 2.2. 域 II：大气层外仿真

适用于穿越或主要运行于大气层外（通常 `h > 86 km` 视为真空边界）的对象，细分为两类：

#### 2.2.1. 运载火箭入轨

- **典型对象**：多级运载火箭（CZ-2F、Falcon 9、SLS 等教学/占位模型）。
- **动力学模型**：
  - **3-DOF 质点 ECI（7 维）**：`[r, v, m]`，主流任务分析。
  - 可选 **6-DOF**：未来扩展姿控、RCS、发动机摆动。
- **关键物理**：
  - 多级并联/串联、助推分离、级间分离、抛整流罩、星箭分离；
  - 推力高度修正、背压损失；
  - J2 引力摄动、地球自转；
  - 开环三段式 / PEG 闭环制导 / 两脉冲圆化；
  - Δv 预算（引力/气动/转向/背压损失）。
- **终止条件**：达到目标轨道能量、SECO、滑行段远地点。
- **核心输出**：入轨轨道根数（a, e, i, Ω, ω, ν）、近地点/远地点、倾角、入轨质量、运载能力。

#### 2.2.2. 洲际弹道导弹（ICBM）

- **典型对象**：远程弹道导弹、潜射弹道导弹。
- **动力学模型**：
  - **动力上升段**：沿用运载火箭 3-DOF ECI + 多级动力。
  - **无动力中段**：`ballistics` 的 ECI 弹道导弹模型（J2 + 扁率 + 阻力）。
  - **再入段**：可切换为 MPM/6-DOF 高保真再入，或使用简化 ECI 再入模型。
- **关键物理**：
  - 主动段程序转弯 / 闭路制导；
  - 中段自由飞行与 J2 摄动；
  - 再入大气、气动减速、热流（Sutton-Graves）、落点精度；
  - 多弹头/MIRV（未来扩展）。
- **终止条件**：落地事件。
- **核心输出**：总射程、再入角、落点、飞行时间、CEP、再入热流峰值、最大减速过载。

#### 2.2.3. 跨大气层统一仿真流程

```
配置 SimConfig
    └─> mission = "rocket" / "icbm" / "suborbital"   # 域 II：大气层外
        ├─> vehicle.type 由 mission 推导
        ├─> environment.atmosphere = 扩展 ISA（0–86 km）+ 86 km 以上指数延拓
        ├─> environment.gravity = 点质量 + J2
        ├─> environment.terrain = 可选（再入用）
        └─> guidance = 开环 / PEG / 弹道整形 / 再入程序

build_phases(cfg)
    └─> [PoweredAscentPhase]   主动段（多级火箭/导弹）
        [CoastPhase]           无动力中段（ICBM）或滑行（火箭圆化）
        [OrbitInsertionPhase]  入轨/SECO（火箭）
        [ReentryPhase]         再入段（ICBM，可选）
        [TerminalPhase]        落地/星箭分离事件

simulate()
    └─> dynamics: powered_eci / point_mass / mpm / sixdof
        事件链：stage_separation / burnout / apogee / orbit_reached / impact_ground

postprocess()
    └─> 轨道根数、射程、落点、再入参数、Δv 预算、事件时间线
```

### 2.3. 两域衔接：出大气层与再入判定

**核心原则**：大气层边界只作为**记录事件**用于可视化分段，**不作为 RHS 模型切换点**，避免在 `solve_ivp` 中引入不连续。

| 事件 | 判定条件 | 处理动作 |
|---|---|---|
| **大气模型衰减** | 大气模型在 `h > 86 km` 后按指数延拓到零 | RHS 中气动阻力/风/动压自然衰减，无需切换；所有 dynamics 一致调用 `atmosphere.density(h)` |
| **出大气层（记录事件）** | `q < q_exit_threshold`（如 0.5 Pa），方向 `direction=-1` | 仅写入 `events_log`，可视化可从此点开始切换配色/轨道线型 |
| **再入大气层（记录事件）** | `q > q_entry_threshold`（如 2 Pa），方向 `direction=+1` | 仅写入 `events_log`；对 ICBM 可从此点启动高保真再入段 |

> **滞回设计**：`q_entry_threshold > q_exit_threshold`（如 2 Pa vs 0.5 Pa），避免飞行器在阈值附近缓慢穿越时反复触发成对事件。
| **落地** | `h ≤ terrain(lon, lat)` 或 `h ≤ 0`（简化模式） | 终止积分，记录落点与状态；地形检测性能优化见 §5.7 |
| **轨道插入** | 能量/速度/高度满足目标轨道 | 终止动力段，转入轨道根数计算或滑行段 |

**为什么不用密度阈值或固定高度硬切换？**
1. 标准大气 86 km 处密度约 `7e-6 kg/m³`，`ρ < 1e-6` 要到更高处才成立，固定高度与密度并用会导致事件触发不一致。
2. 火箭真正“感受不到大气”的判据是**动压 `q`** 低于阈值，而非高度或密度绝对值。
3. 在 Phase 边界关闭气动阻力会让 RHS 不连续，变步长积分器会反复缩步甚至失败；让大气模型平滑衰减到零是更数值稳定的做法。
4. ICBM 中段会二次穿越边界，事件必须带 `direction` 参数；出/再入分别用 `-1`（动压由大变小）和 `+1`（由小变大）。

- 建议在 `Phase` 切换时显式记录“出大气层”“再入”“级间分离”等事件到 `events_log`，便于可视化分段着色。
- 对 ICBM，主动段结束时的位置速度直接作为中段初始条件；中段结束（再入点）的状态投影到 MPM/6-DOF 的 ENU 初值，实现 fidelity 升降。详见 §6.4 状态切换策略。


## 3. 两个源项目源码级对比

### 3.1. `ballistics`（外弹道方程）

- **规模**：~9,663 行核心代码，343 个测试，111 个已追踪文件。
- **模型谱系**：
  - 修正质点模型 MPM（8/13 维）
  - 6-DOF 刚体模型（13 维，四元数弹轴）
  - ECI 弹道导弹模型（B1 无动力、B2 多级动力、B3/B4 弹道整形/火控、B5 再入加热）
- **环境模型**：扩展 ISA、WGS84 重力、Dryden 阵风、GRIB2/SRTM/GeoTIFF 地形。
- **入口**：CLI（`run_sim.py`）、Tkinter GUI（`launch_gui.py`）、FastAPI（`api/main.py`）、Web（`web/`）。
- **装配层**：`config.py` 统一组装 `RunConfig`，被 CLI/API/GUI 复用。
- **可视化**：`ballistics/viz/` 纯函数 matplotlib，Three.js Web 3D。
- **强项**：
  - 模型保真度层次丰富；
  - 测试与射表验证体系完整；
  - 配置/后处理/可视化三端复用设计优秀。
- **弱项/局限**：
  - 版本号不一致（`__init__.py` 0.2.0 vs `pyproject.toml` 0.7.0）；
  - 6-DOF 火控数值噪声较大，**部分工况存在发散风险**；
  - GPU/批量后端物理简化，存在系统偏差；
  - 远程地球物理在 MPM/6-DOF 中仍是局部近似。

> **迁移前必须完成**：在阶段 0 之前，先在 `ballistics` 源项目中定位并文档化 6-DOF 的发散工况（即便不修复，也要形成已知失效清单）。迁移后若再遇到发散，才能分清是搬运引入的 bug 还是原有问题。

### 3.2. `lvsim`（火箭仿真程序）

- **规模**：30+ 测试文件，核心包结构清晰。
- **模型谱系**：
  - 3-DOF 质点模型（7 维：`r, v, m`）
  - 多级火箭阶段：助推 + 芯级 + 上面级 + 抛罩 + 星箭分离
  - 可选 PEG 闭环制导、两脉冲圆化
- **环境模型**：扩展 ISA、点质量+J2 引力、参数化风廓线。
- **入口**：CLI（`run_sim.py`）、菜单启动器（`launch.py`）、Tkinter GUI（`launch_gui.py`）、`启动.bat`。
- **装配层**：`config.py` 组装 cfg + 校验/摘要；`phases.py` 生成阶段与事件链。
- **可视化**：`lvsim/viz/` 丰富静态图 + GUI 统一播放时钟 + 三维动画 GIF/MP4。
- **强项**：
  - 阶段驱动架构天然适合多级火箭；
  - 制导律（开环/PEG）与自动配平成熟；
  - GUI 播放时钟与动画体验好；
  - 零件库 + 自定义装配扩展性强。
- **弱项/局限**：
  - 纯 3-DOF，无 6-DOF 姿态/力矩；
  - 无高保真炮弹/导弹阻力模型（G1/G7）；
  - 风模型较简单，无地形落地检测；
  - 默认三级为“惰性上方质量”，需显式追加真三级。

### 3.3. 互补性结论

| 能力 | `lvsim` | `ballistics` | 融合价值 |
|---|---|---|---|
| 多级动力（火箭/ICBM） | ✅ 强 | ⚠️ 有但较薄 | 以 lvsim 为主 |
| 无动力弹道 | ⚠️ 弱 | ✅ 强 | 以 ballistics 为主 |
| 6-DOF 姿态 | ❌ 无 | ✅ 强 | 引入 ballistics |
| 阻力模型 | ⚠️ Cd-Ma 表 | ✅ G1/G7/自定义 | 引入 ballistics |
| 风场 | ⚠️ 廓线 | ✅  Dryden/GRIB2/3D | 引入 ballistics |
| 地形落地 | ❌ 无 | ✅ 地形/SRTM | 引入 ballistics |
| 火控反解 | ❌ 无 | ✅ 有 | 引入 ballistics |
| 再入加热 | ❌ 无 | ✅ Sutton-Graves | 引入 ballistics |
| 入轨制导（PEG） | ✅ PEG | ⚠️ 有但薄 | 以 lvsim 为主 |
| 弹道整形/ICBM 瞄准 | ❌ 无 | ✅ 有 | 引入 ballistics |
| 可视化 GUI | ✅ 播放时钟 | ✅ 三端复用 | 取长补短 |

## 4. 总体架构设计

### 4.1. 目录结构（目标态）

### 4.2. 统一配置层 `config.py`

统一配置层采用 **pydantic BaseModel**（核心依赖之一），兼顾类型安全、内置校验与 API 友好：

```python
from pydantic import BaseModel, Field

class SimConfig(BaseModel):
    mission: str = Field(..., pattern=r"^(projectile|missile|rocket|icbm|suborbital)$")
    vehicle: VehicleConfig           # 飞行器装配（类型由 mission 推导）
    launch: LaunchConfig             # 发射点/时间/方向
    environment: EnvironmentConfig   # 大气/风/重力/地形
    guidance: GuidanceConfig         # 制导律参数
    options: OptionsConfig           # 积分器、事件、模型开关、输出选项、可视化开关
```

- `VehicleConfig` 支持三种装配模式，按 `mission` 自动选择：
  - **弹丸/导弹模式**（大气层内）：质量/直径/阻力律/稳定因子/发动机。
  - **火箭/ICBM 模式**（大气层外）：零件库组合（boosters/stages/fairing/payload）。
  - **混合模式**（未来扩展）：火箭增程弹、制导炮弹等。
- 配置校验：pydantic 负责字段类型/范围校验；`validate_config()` 负责跨字段业务校验（如 T/W>1、推进剂守恒、倾角可达），输出 FAIL/WARN 列表。
- 覆盖语法：支持点路径 `overrides`，如 `"guidance.kick_deg": 15.0`。实现方式：
  ```python
  def apply_overrides(cfg: SimConfig, overrides: dict) -> SimConfig:
      # 将点路径展开为嵌套 dict，再用 pydantic model_copy(update=...)
      ...
  ```

### 4.3. 统一仿真器 `simulator.py`

以 `lvsim` 的 `simulate()` 多阶段循环为基础，抽象为 **Phase 列表驱动**：

```python
def simulate(cfg: SimConfig) -> SimResult:
    phases = build_phases(cfg)
    state = initialize_state(cfg)
    trajectory = []
    events_log = []
    for phase in phases:
        sol = solve_ivp(
            fun=phase.rhs,
            t_span=phase.t_span,
            y0=state,
            events=phase.events,
            args=(cfg, phase),
            method=cfg.options.integrator,
            dense_output=True,
        )
        trajectory.append(sol)
        events_log.extend(phase.process_events(sol))
        state = phase.final_state(sol)
        if phase.is_terminal:
            break
    return postprocess(trajectory, events_log, cfg)
```

每个 `Phase` 决定：
- 使用的运动方程（point_mass / mpm / sixdof / powered_eci）
- 推力/质量变化（动力段）
- 制导律（开环/PEG/火控）
- 终止事件（燃尽/分离/落地/远地点）

这样既能保留 lvsim 的火箭阶段链，也能插入 ballistics 的无动力/再入段。

## 5. 关键模块融合方案

### 5.1. 坐标系与参考框架

**原则：统一转换层，而非统一坐标系。** 不同 dynamics 模块根据其物理尺度与精度需求选择原生坐标系；所有转换函数统一放在 `frames.py`。

| 域 | dynamics | 原生坐标系 | 理由 |
|---|---|---|---|
| 域 I（大气层内） | `mpm.py`、`sixdof.py` | **ENU**（发射点东北天） | 近程炮弹/导弹位移在百米~百公里量级，ENU 避免地心坐标 `6.4e6 m` 吃掉米级精度；科氏力作为显式摄动项加入，自转项按需启用。 |
| 域 II（大气层外） | `point_mass.py`、`powered_eci.py` | **ECI**（J2000 近似） | 火箭/ICBM 跨越大尺度，ECI 最自然；地球自转通过 ECI↔ECEF 转换处理。 |
| 通用 | — | **ECEF / LLA** | 地面轨迹、着陆点、地形查询、可视化标注。 |

- 复用 `lvsim/frames.py` 的 ECI↔ECEF↔ENU 转换，补充 `ballistics/coordinates.py` 的 Bowring 大地高算法。
- 每个 `dynamics` 模块在 `__init__` 或文档字符串中显式声明：`native_frame`、`state_vector`、`input_units`、`output_units`。
- `simulator.py` 在阶段切换时负责坐标转换，而非要求每个 RHS 内部转换。
- 统一约定：**内部全 SI + 弧度**；对外接口保留度数/千米等友好单位。

### 5.2. 大气模型

- **主体**：`lvsim/atmosphere.py` 的扩展 ISA（0–86 km）。
- **增强**：从 `ballistics/atmosphere.py` 引入湿度虚温修正、自定义探空廓线。
- **接口**：
  ```python
  class AtmosphereModel(Protocol):
      def density(self, h_m: float) -> float: ...
      def pressure(self, h_m: float) -> float: ...
      def temperature(self, h_m: float) -> float: ...
      def sound_speed(self, h_m: float) -> float: ...
  ```

### 5.3. 引力模型

- **主体**：`lvsim/gravity.py` 的点质量 + J2。
- **增强**：从 `ballistics/gravity.py` 引入 WGS84 Somigliana 正常重力，用于近地 MPM/6-DOF 模式做快速校核。
- 默认使用 J2 模型，模式开关 `gravity="simple"` 用于教学。

### 5.4. 阻力/气动模型

- **火箭模式**：保留 `lvsim/aerodynamics.py` 的 `Cd-Ma` PCHIP 插值，但扩展为可读取自定义表。
- **弹丸/导弹模式**：引入 `ballistics/drag.py` 的 G1/G7 标准阻力律与自定义阻力系数表。
- **6-DOF 模式**：直接迁移 `ballistics/sixdof.py` 的气动力矩模型。
- 统一设计 `AeroModel` 协议，支持：
  - `drag_coefficient(mach, alpha)`
  - `lift_coefficient(mach, alpha)`（用于 MPM/6-DOF）
  - `force_moment(state, env)`（用于 6-DOF）

### 5.5. 推进与质量模型

- **主体**：完全迁移 `lvsim/propulsion.py`。
- `ThrustModel` 支持：
  - 真空推力 + 高度背压修正（当前）
  - 节流阀/推力曲线（未来扩展）
  - 比冲随高度/混合比变化（未来扩展）
- `Stage` 对象沿用 lvsim 的零件库，但增加 `aero_model` 字段，使同一级可挂载不同气动数据库。

### 5.6. 风模型

- **主体**：以 `ballistics/wind.py` 为基准。
- 支持：
  - 均匀风、对数/幂律廓线
  - Dryden 阵风（距离索引）
  - 自定义探空廓线
  - 3D 风场 + 地形梯度修正
  - GRIB2 数值天气预报接入（可选）
- 在火箭模式中，风只改变相对速度；在弹丸/导弹模式中，风同时影响 MPM 的横偏与 6-DOF 的侧滑。

### 5.7. 地形与落地检测

- **主体**：迁移 `ballistics/terrain.py`。
- 支持：
  - 程序化地形
  - 灰度图/GeoTIFF DEM
  - SRTM `.hgt` 读取
- 落地事件统一使用几何高度 ≤ 地形高，替代当前 lvsim 的 `h <= 0`。
- **性能优化**：
  - DEM 在 `TerrainModel` 初始化时一次性加载到内存，并预计算双线性插值系数；
  - 落地事件采用两层检测：粗检测用发射点高程平面（快速），事件触发后再在局部用真实地形做精化；
  - 对灰度图/GeoTIFF 做瓦片缓存，只加载弹道走廊附近的瓦片；
  - 提供 `terrain = null` 模式，完全关闭地形查询，用于快速批量计算。
- 这对弹道导弹再入、亚轨道火箭回收、炮弹落点分析至关重要。

### 5.8. 制导与控制

| 模式 | 来源 | 用途 |
|---|---|---|
| 开环三段式 | lvsim | 火箭垂直起飞 → 程序俯仰 → 重力转弯 |
| 上面级压平 | lvsim | 上面级线性压平到水平 |
| PEG 闭环 | lvsim | 真空段能量到点迭代，命中轨道 |
| 两脉冲圆化 | lvsim | 滑行 + 远地点切向脉冲 |
| 火控反解 | ballistics | 反解射角/方位命中地面目标 |
| 弹道整形 | ballistics | 最小能量/高抛/压低 |
| 6-DOF 控制 | ballistics | 鸭舵/脉冲发动机控制力矩 |

- 设计 `GuidanceLaw` 协议，`Phase` 根据当前段选择制导律。
- 对弹道导弹，动力上升段用 PEG/开环，无动力中段用弹道整形，再入段用预置攻角程序。

### 5.9. 事件系统

统一事件检测函数：

```python

def event_apogee(t, y, cfg, phase): ...
def event_burnout(t, y, cfg, phase): ...
```

## 6. 运动方程层设计

### 6.1. 统一状态向量规范

为简化多阶段切换，定义不同 fidelity 的状态包装，但内部保持为 NumPy 1D 数组：

| 模型 | 状态向量 | 维度 |
|---|---|---|
| 3-DOF 质点 | `[x, y, z, vx, vy, vz, m]` | 7 |
| MPM | `[E, N, U, vE, vN, vU, spin, alpha]`（可扩展 13 维动态偏航） | 8/13 |
| 6-DOF | `[r(3), v(3), q(4), omega_y, omega_z, p]` | 13 |
| 弹道导弹 ECI | `[r(3), v(3), m]` | 7 |

阶段切换时提供 `project_state()` 函数做 fidelity 升降维：
- 6-DOF → 3-DOF：丢弃姿态与角速度，保留质量。
- 3-DOF → MPM：转换到 ENU，估算自转。
- 动力段 → 无动力段：保留位置速度，更新质量。

### 6.2. 运动方程实现策略

- `point_mass.py`：合并 `lvsim/dynamics.py` 与 `ballistics` 中用于 ECI 的质点方程。
- `mpm.py`：直接迁移 `ballistics/mpm.py`，增加 `AtmosphereModel`/`WindModel` 注入。
- `sixdof.py`：直接迁移 `ballistics/sixdof.py`，推力模型替换为新 `ThrustModel`。
- `powered_eci.py`：在 `lvsim/dynamics.py` 基础上，集成 `ballistics` 的更精细阻力与风模型。

### 6.3. 积分器与精度

- 默认 `scipy.integrate.solve_ivp` + `DOP853`（高保真）/ `RK45`（快速）。
- **蒙特卡洛默认使用与单发完全相同的模型和积分器**，保证统计分布无系统偏差；batch/GPU 固定步长 RK4 仅作为可选加速后端。
- 对 batch/GPU 后端，必须提供“batch vs 单发抽样对拍”测试，量化并文档化系统偏差（例如落点均值/标准差差异 <1%）。
- 事件检测密集化输出由 `dense_output=True` 支持。

### 6.4. 状态切换策略（fidelity 升降维与坐标系转换）

阶段切换时由 `simulator.py` 调用 `project_state()` 完成状态映射。**降维是良定义的，升维是病态的，必须显式规定策略与禁止项。**

#### 6.4.1 允许的切换方向

| 源段 | 目标段 | 类型 | 策略 |
|---|---|---|---|
| 6-DOF | 3-DOF | 降维 | 丢弃姿态四元数与角速度，保留 `r, v, m`；能量、动量守恒自动保持。 |
| 3-DOF ECI | 3-DOF ENU | 同维换系 | ECI→ECEF→ENU 转换；速度需扣除地球自转牵连项。 |
| 3-DOF ENU | MPM | 升维 | 位置速度直接映射；`spin` 由膛线缠角或末级自旋率给定；`alpha` 初始化为零攻角配平。 |
| 3-DOF ECI | MPM | 跨域升维 | 先 ECI→ENU，再执行上一条；需指定发射方向北/东分量。 |
| 6-DOF | MPM | 降维 | 用当前弹轴方向估算攻角 `alpha`，`spin` 取当前 `p`。 |
| 动力段 | 无动力段 | 同维 | 保留 `r, v, m`，质量按阶段配置更新。 |
| 无动力段 | 动力段 | 同维 | 保留 `r, v`，质量按新级干重 + 推进剂初始化。 |

#### 6.4.2 禁止或高风险的切换

| 切换 | 风险 | 处理 |
|---|---|---|
| MPM/6-DOF ENU → 3-DOF ECI | 无全局位置信息，无法定义 ECI 坐标 | 禁止；必须先在 ENU 中给出发射点 LLA，再转换。 |
| 3-DOF → 6-DOF | 需构造四元数、角速度，信息不存在 | 仅允许在“再入起始”等用户明确给出初始姿态的场合使用；默认禁止自动升维。 |
| 任意升维 → 能量/动量跳变 | 构造的新状态可能与原状态能量不一致 | 每次升维后计算 `delta_E`、`delta_p`，写入日志；若超过阈值则报警。 |

#### 6.4.3 再入段升维示例（ICBM）

从 3-DOF ECI 切到 6-DOF ENU：位置速度直接映射，弹轴取速度方向（零攻角配平），滚转角任意置零，自旋由弹头/末级 spin 率给定。

```python
def reentry_to_sixdof(eci_rv: np.ndarray, site: LaunchSite, cfg: ReentryConfig) -> np.ndarray:
    """将 ECI 再入点状态投影为 6-DOF 的 ENU 初值。
    返回布局：[r(3), v(3), q(4), omega_y, omega_z, p]
    """
    # 1. ECI -> ECEF -> ENU
    enu_pos, enu_vel = eci_to_enu(eci_rv[:3], eci_rv[3:6], site)

    # 2. 弹轴 x_b 与速度方向对齐，滚转角置零
    v_dir = enu_vel / np.linalg.norm(enu_vel)
    # rotation from body x-axis [1,0,0] to v_dir
    q = rotation_quaternion(from_vec=np.array([1.0, 0.0, 0.0]), to_vec=v_dir)

    # 3. 自旋率由配置给定；横向角速度初始为零
    spin_rate = cfg.spin_rate_rps
    omega_y, omega_z = 0.0, 0.0

    # 4. 一致性检查：ENU -> ECI -> ENU 往返，位置/速度相对误差应极小
    eci_back = enu_to_eci(enu_pos, enu_vel, site)
    pos_err = np.linalg.norm(eci_back[:3] - eci_rv[:3]) / np.linalg.norm(eci_rv[:3])
    vel_err = np.linalg.norm(eci_back[3:] - eci_rv[3:]) / np.linalg.norm(eci_rv[3:])
    if pos_err > 1e-9 or vel_err > 1e-9:
        logger.warning("ECI↔ENU 往返误差过大: pos_err=%.3e, vel_err=%.3e", pos_err, vel_err)

    # 5. 速度模长检查：ENU 速度已扣除地球自转牵连项，应与 ECI 速度模长不同，
    #    但同一参考系内降维/升维后速度模长必须保持不变。
    speed_eci = np.linalg.norm(eci_rv[3:6])
    speed_enu = np.linalg.norm(enu_vel)
    logger.info("再入升维: ECI speed=%.1f m/s, ENU speed=%.1f m/s (delta=%.1f)",
                speed_eci, speed_enu, speed_eci - speed_enu)

    return np.concatenate([enu_pos, enu_vel, q, [omega_y, omega_z, spin_rate]])
```

> **若目标是 MPM 而非 6-DOF**：返回 `[E, N, U, vE, vN, vU, spin_rate, alpha]`，其中 `alpha` 初始化为 0（零攻角配平）。位置速度部分与上面完全相同。

#### 6.4.4 单元测试要求

**同维换系测试（ECI ↔ ENU）**：
- 随机生成 ECI 状态，执行 `ECI → ENU → ECI` 往返；
- 位置相对误差 `<1e-9`，速度相对误差 `<1e-9`。

**升降维测试（3-DOF → MPM/6-DOF）**：
- 输入输出 `r, v, m` 分量严格相等；
- 速度模长严格不变；
- 新增自由度单独断言：
  - MPM：`spin` 等于给定值，`alpha == 0`；
  - 6-DOF：弹轴四元数与速度方向对齐（`q * [1,0,0]` 与 `v/|v|` 点积 > 0.999999），`ωy == ωz == 0`，`p` 等于给定值。

**禁止切换测试**：
- 每种禁止方向写一个“应抛出 `NotImplementedError` 或 `ValueError`”的测试。

**ICBM 端到端测试**：
- 主动段 → 中段 → 再入段全程，检查同一坐标系下的比能量曲线单调递减且无不合理跳变。

### 7.6. 可视化效果标准

1. **正确性**：所有可视化必须基于真实仿真结果，不得使用占位数据；关键标注（顶点、落点、SECO）与事件日志一致。
2. **一致性**：同一物理量在不同图中单位、颜色、标签一致；对外坐标轴优先使用友好单位（km, km/h, °C, g0）。
3. **信息密度**：每张图聚焦一个主题，Dashboard 类可组合但不超过 3×3 子图；避免过度绘制。
4. **可导出性**：所有图支持导出矢量/栅格；动画支持自定义帧率与分辨率。
5. **可测试性**：每个 `viz/` 函数提供“冒烟测试”：给定最小结果对象，返回非空 `Figure` 且不抛异常。
6. **性能**：静态图生成时间 < 2 s；GUI 播放时钟在 4× 倍速下不卡顿。
7. **Web 3D 性能**（第二阶段）：Three.js 在 10k 轨迹点内保持 30 FPS。

## 8. 测试与验证策略

### 8.1. 测试分层

1. **单元测试**：大气、引力、阻力、坐标系、推进、事件函数。
2. **模型一致性测试**：
   - MPM vs 3-DOF 在简化条件下结果一致；
   - 新 simulator vs lvsim 的 CZ-2F 全程（误差 <1%）；
   - 新 simulator vs ballistics 的 M107 射表（误差 <2%）。
3. **集成测试**：火箭动力上升 → 无动力弹道 → 落地/入轨完整链路。
4. **可视化冒烟测试**：所有 viz 函数能生成非空 Figure。
5. **GUI 接线测试**：参数面板切换、播放按钮、画布刷新。
6. **回归测试**：保留两个源项目的关键测试用例。

### 8.2. 验证基线

- **火箭基线**：复现 lvsim 的 CZ-2F 入轨结果，对比近地点、远地点、倾角；检查 globe3d/orbit_insertion/dvbudget 可视化正确。
- **弹丸基线**：复现 ballistics 的 M107 射表，对比最大射程、落角、飞行时间；检查 trajectory3d/groundtrack/montecarlo 可视化正确。
- **ICBM 基线**：组合动力上升 + 无动力中段 + 再入，检查总射程与公开数据量级一致；验证跨大气层剖面图与再入热流图。

### 8.3. 代码质量工具

- `black` 行宽 100
- `flake8`
- `mypy`
- `pre-commit`
- 覆盖率目标 ≥80%（核心物理 ≥90%）

### 8.4. 迁移对拍脚本（Regression Bridge）

迁移期间最强的回归防线：**黄金文件（golden file）方案**。本地用旧项目生成基准结果并提交进新仓库，CI 只对比新平台输出与黄金文件，避免 CI 依赖旧项目环境。

**黄金文件生成（本地一次性）**：
```bash
# 在旧项目环境中运行
python scripts/generate_golden.py \
    --old-project D:/Pycharm/游戏与兴趣/外弹道方程 \
    --output tests/golden/atmospheric_m107.json
python scripts/generate_golden.py \
    --old-project D:/Pycharm/游戏与兴趣/火箭仿真程序 \
    --output tests/golden/exo_cz2f.json
```

每个黄金文件保存：
- 配置快照；
- 关键标量（射程、落角、飞行时间、近地点、远地点、倾角、事件时间等）；
- 关键时历采样（t, h, v, mass, q, Mach 等，降采样到 ~500 点）。

**对拍脚本（本地/CI 两用）**：
```bash
python scripts/regression_bridge.py --golden tests/golden/atmospheric_m107.json --preset M107
python scripts/regression_bridge.py --golden tests/golden/exo_cz2f.json --rocket CZ-2F
```

**输出**：
- 差异表（绝对误差、相对误差、是否超阈值）；
- 生成 `regression_report.md`，超阈值时 CI 失败并高亮差异最大的列。

**更新黄金文件的纪律**：
- 仅当确认旧项目本身存在 bug、且新平台修复正确时，才更新黄金文件；
- 更新必须通过 PR，并在 `CHANGELOG.md` 中说明理由。

## 12. 风险与注意事项

| 风险 | 影响 | 缓解措施 |
|---|---|---|
| 两个项目依赖冲突 | 高 | 新建虚拟环境，统一 `pyproject.toml`，分阶段解决导入错误；重依赖拆分为 extras |
| 物理单位不统一 | 高 | 制定并文档化内部 SI + 弧度约定；新增单位转换层 |
| 坐标系误用 | 高 | 每个 dynamics 声明原生坐标系；域 I 用 ENU，域 II 用 ECI；转换函数统一在 `frames.py` |
| 状态向量维度切换复杂 | 中 | 严格定义 `project_state()`，显式允许/禁止列表；升维后做能量/动量守恒检查 |
| 6-DOF 发散 | 中 | 迁移前普查并文档化已知失效工况；6-DOF 默认仅在稳定工况启用 |
| 地形落地事件性能差 | 中 | DEM 内存缓存 + 两层检测（粗平面 + 精地形）+ `terrain = null` 快速模式 |
| 蒙特卡洛系统偏差 | 中 | MC 默认走单发模型；batch/GPU 作为可选加速，必须通过对拍量化偏差 |
| 可视化风格不统一 | 低 | 统一配色与标签规范，分阶段迁移 |
| 性能下降 | 中 | 保留 batch/GPU 路径作为可选后端；先保证正确性再优化 |
| 许可证/版权问题 | 低 | 两个项目均为用户自有代码；阶段 0 选定 MIT/BSD-3-Clause 并写入仓库 |
| Git 大文件问题 | 中 | `.gitignore` 排除缓存、动画、DEM 数据；文档说明如何下载 |

## 13. 结论

将 `lvsim` 的**阶段驱动火箭仿真骨架**与 `ballistics` 的**高保真外弹道物理库**结合，按**大气层内**与**大气层外**两大域进行统一建模，是技术上高度互补、产品上顺理成章的选择。最终目标是一个覆盖“炮弹—战术导弹—洲际导弹—运载火箭”全谱系的统一弹道模拟程序，具备 CLI/GUI/API 三入口、分域定制的丰富可视化（轨迹、轨道、落点、再入、动画）与完整的测试验证体系。

建议按本方案的 6 个阶段推进，每个阶段都保持**可运行、可测试、可回退**，确保最终能稳定推送到 `https://github.com/bowlbowlhe-svg/ballistic-sim`。
